# 测试 Prompt

复制以下内容发送给 AI 测试助手：

---

你是一名软件测试工程师。你需要对「记账APP」这个项目进行全面的功能测试。

## 你的任务

1. **阅读项目**：先通读 CLAUDE.md 了解产品全貌，再浏览 src/ 目录下的源码理解实现细节
2. **自主设计测试用例**：根据你对项目的理解，自己设计测试场景，覆盖：
   - 所有页面的核心功能
   - 边界情况（空数据、极端数值、异常输入）
   - 多页面之间的数据联动（如花销入库后欠款是否同步）
   - 数据持久化（关闭重开数据是否还在）
   - 交互体验问题
3. **执行测试**：运行 `npm run dev` 启动 APP，逐一执行你设计的测试用例
4. **报告问题**：记录发现的所有问题，按严重程度排序（阻断 > 严重 > 一般 > 建议）

## 测试要求

- 不要问我应该测什么，你自己想
- 不要拘泥于文档里写的功能，去探索文档没提到的角落
- 尝试用各种奇怪的输入和操作来"搞坏"这个 APP
- 每个问题写清楚：复现步骤 → 预期行为 → 实际行为 → 截图

## 启动方式

```bash
npm install
npx @electron/rebuild
npm run dev
```
