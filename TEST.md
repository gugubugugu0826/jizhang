# 记账APP — 测试文档

## 环境准备

1. 确保电脑已安装 Node.js（18+），如未安装：https://nodejs.org
2. 打开终端，进入项目目录，执行：
   ```bash
   npm install
   npx @electron/rebuild
   npm run dev
   ```
3. APP 窗口自动打开

---

## 测试用例

### 用例 1：AI 智能录入（核心功能）

1. 点击左侧「⚡ 智能录入」
2. 点击右上角「配置 AI」，输入 DeepSeek API Key
3. 粘贴以下测试文字：

```
6月15号打车13.75刀 四个人平分
6月11号麦当劳 总共43.78刀 黄柏清15.05刀 廖泽平7.1刀加16.25刀
5月26号有个180人民币的UberEat 张先澍28.8刀 廖泽平21.8刀加3.8刀
6月17号披萨四个人平分56.93刀
6月18号汉堡王 廖泽平和刘承洲平分50.2刀
6月20号麦当劳 廖泽平16.25刀加6.7刀 黄柏清和刘承洲16.8刀 总花费45.32刀
6月16号收卡 廖泽平和张先澍评分368.6人民币
```

4. 点击「AI 解析」
5. **预期结果**：等待 3-10 秒，弹出结果窗口，显示解析出的多笔记录
6. **检查项**：
   - 每笔的币种（CNY/AUD）是否正确
   - 每笔的分类是否匹配（打车→交通，麦当劳→餐饮，UberEat→餐饮）
   - 分摊人是否正确识别（名字变体如"黄百清"→"黄柏清"也应匹配）
   - 底部「本次分账统计」显示各人汇总金额
7. 勾选要入库的记录 → 点「确认入库」
8. **预期结果**：提示入库成功，弹窗关闭

---

### 用例 2：手动添加花销

1. 进入「💰 花销记录」
2. 点击「添加花销」
3. **检查项**：
   - 币种可选择 AUD/CNY
   - 分类有 10 个一级分类，二级分类随一级联动
   - 日期默认今天
4. 填写金额（如总金额 100 刀），选分类
5. 在「分摊明细」区，点击「四人平分」→ 自动分配 4 人各 25 刀
6. 点击「确认添加」
7. **预期结果**：列表出现新记录，币种、分摊人、金额正确

---

### 用例 3：花销记录管理

1. 在「💰 花销记录」页
2. **筛选测试**：
   - 选币种为 AUD → 只显示澳元记录
   - 选某个人员 → 只显示该人参与分摊的记录
   - 选分类 → 只显示该分类
   - 选日期范围 → 只显示范围内的
3. **多选删除测试**：
   - 勾选 2-3 条记录
   - 顶栏出现「已选 N 条」→ 点「删除选中」
   - **预期结果**：选中的记录被删除
4. **一键删除测试**：
   - 点表格底部红色「一键删除所有」
   - 弹确认框 → 点确定
   - **预期结果**：所有记录被删除

---

### 用例 4：分账结算

1. 先通过智能录入添加几条包含 CNY 和 AUD 的花销
2. 进入「💲 分账结算」
3. 选日期范围覆盖刚添加的记录
4. 点刷新
5. **预期结果**：
   - 顶部两个卡片分别显示人民币总计和澳元总计
   - 表格列出每人的人民币金额和澳元金额，分开显示，不做换算
   - 底部合计行正确

---

### 用例 5：欠款结余

1. 进入「📒 欠款结余」
2. **预期结果**：
   - 左边 CNY 栏和右边 AUD 栏分别显示每人当前欠款余额
   - 红色正数 = 对方欠我，绿色负数 = 我欠对方，灰色 0 = 已结清
   - 余额应与之前录入的花销分摊金额一致
3. **手动记账测试**：
   - 点「手动记账」
   - 选人「刘承洲」→ 类型「对方欠我」→ AUD → 金额 300 → 原因写「借款」→ 确认
   - **预期结果**：AUD 栏刘承洲余额增加 +300 刀
4. **清账测试**：
   - 点击某人的「清账」按钮
   - **预期结果**：该人该币种余额变为 0（已结清）

---

### 用例 6：变动记录

1. 进入「📜 变动记录」
2. **预期结果**：能看到之前操作产生的所有记录：
   - 花销分摊（标记「花销分摊」）
   - 手动记账（标记「手动记账」）
   - 清账（标记「清账」）
3. **筛选测试**：分别按人员、币种、类型筛选
4. **删除测试**：删除一条记录后，回到欠款结余页看余额是否相应变化

---

### 用例 7：统计分析

1. 进入「📊 统计分析」
2. 选币种筛选
3. **检查项**：
   - 左边饼图：支出分类占比
   - 中间饼图：人员支出占比
   - 右边柱状图：月度趋势
   - 顶部汇总金额正确
4. 选日期范围后点刷新，图表应相应变化

---

### 用例 8：人员管理

1. 进入「👥 人员管理」
2. 添加新人员「测试员」
3. 编辑「测试员」改名为「临时人员」
4. **预期结果**：智能录入和添加花销的分摊人选里能看到新人名
5. 删除「临时人员」

---

### 用例 9：AI 配置

1. 进入「⚙️ 设置」
2. **汇率测试**：修改 AUD→CNY 汇率为 5.0，保存，刷新后确认显示新汇率
3. 进入智能录入 → 配置 AI → 切换到「OpenAI」
4. 输入 OpenAI API Key → 保存
5. **预期结果**：智能录入页显示「OpenAI AI 已就绪」
6. 切换回 DeepSeek，确认正常

---

### 用例 10：本地解析

1. 进入智能录入，清空 AI Key（或故意输错）
2. 粘贴测试文字，点「本地解析」
3. **预期结果**：使用本地规则解析，可能有部分记录缺分摊人（标橙色），识别率低于 AI 但可手动补充

---

## 验证要点

| 检查项 | 内容 |
|--------|------|
| 数据一致性 | 花销入库后，欠款余额自动更新；清账后余额归零 |
| 多币种隔离 | CNY 和 AUD 在分账结算、欠款结余中完全分开 |
| 人名容错 | AI 解析能将打字变体（黄百清→黄柏清）正确匹配 |
| 单条/批量删除 | 花销记录支持单删、多选删、一键全删；变动记录支持单删 |
| 持久化 | 关闭 APP 后重新打开，数据不丢失 |
